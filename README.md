## サービス概要
## どんなサービスなのかを３行で説明してください。
テイクアウト料理店で並ばなくても注文できるアプリ。
客側からの発注、店側の受注処理はもちろん、
料理が完成したら店側から通知が来る。

## このサービスへの思い・作りたい理由
## このサービスの題材となるものに関してのエピソードがあれば詳しく教えてください。
## このサービスを思いつくにあたって元となる思いがあれば詳しく教えてください。
某有名なファストフード店を利用しております。
私はその店の公式アプリを用いて、並ばずに注文しているのですが、注文を受け取る際には店の中で受け取る必要があります。
しかし、店全体の注文状況によっては、遅延が発生することもあります。
受け取りまでの時間を待機したり、受け取りまでについでに用事を済ませたいと思うときがあり、受け取り可能時間が分からないことが意外と煩わしく思っています。
そこで、完成したことを通知できる機能があれば便利なのではないかと考えました。

## ユーザー層について
## 決めたユーザー層についてどうしてその層を対象にしたのかそれぞれ理由を教えてください。
①スマホでの注文機能が無いテイクアウト店
②(ドライブスルーを除く)テイクアウトをよく利用し、待機時間が煩わしいと思っているユーザー(特にタイムパフォーマンスを意識しやすい20代)

①については、逆に公式アプリなどのスマホでの注文機能がある店は、新しいアプリに切り替えるのがめんどくさいのではないかと考える。どちらかというと、スマホでの注文機能を実装していないテイクアウト店向けにしたい。
②については、私と同じように受け取り時間が煩わしいと思っている人に使ってほしいですね。

## サービスの利用イメージ
## ユーザーがこのサービスをどのように利用できて、それによってどんな価値を得られるかを簡単に説明してください。
客側は既に世に出回っているテイクアウトできるアプリのように発注できます。加えて完成通知機能を受け取ることで、商品受け取りまでの時間を今までよりも自由に生かせるようになります。
気がかりなことは店側の都合で、完成品の通知をしなければいけないことです。受付係は今までの作業に加えて、品物が完成したら通知するという作業が加わるわけですから、ひと手間かかる上に、通知を忘れてしまったら、このアプリの利点を生かせなくなってしまいます。通知機能はなるべく手軽な操作でできるように実装し、なるべく手間にならないようにできればと思いますが、通知忘れについては受付係の責任になってしまうでしょう。
